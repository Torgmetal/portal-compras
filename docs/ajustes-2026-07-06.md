## ⚠️ Importante pro dev (leia antes de mexer)

1. **Schema alterado por SQL cru, NÃO por migration.** Estas tabelas/colunas foram
   criadas com `CREATE TABLE`/`ALTER TABLE` direto no Neon e o `schema.prisma` foi
   atualizado pra bater. **Não existe migration** pra elas:
   - `MuralAviso` (tabela nova) — mural de comunicados
   - `FeedbackRH` (tabela nova) — sugestões do funcionário
   - `ContaPagar.categoriasRateio` (coluna `Json?`) — rateio de categorias do Omie

   → Se rodar `prisma db push`, ele vai acusar "drift" e querer **dropar** coisas
   (`Ferias.descontos`, `PontoItem.empresa` etc.). **NÃO aceite** — essas colunas
   já estão no schema deployado; o "drift" é falso (cópia local desatualizada via
   iCloud). Quando precisar de coluna nova, use `ALTER TABLE` + ajuste o schema, ou
   alinhe comigo antes de um `db push`.

2. **`statusEntrega` tem 2+ valores.** Recebimento manual grava `"RECEBIDO"`; o
   auto-sync grava `"ENTREGUE"/"ATRASADO"` (+ `PARCIAL`). Sempre checar o conjunto
   `["ENTREGUE","ATRASADO","RECEBIDO"]` (+ PARCIAL) — nunca só `=== "RECEBIDO"`.

3. **Omie rate-limit.** Os `omie()` de `lib/omie-contas-pagar.js` e `lib/omie.js`
   agora fazem retry respeitando o "Aguarde N segundos" (erro *Consumo redundante*).
   Isso evita o cron morrer no meio do sync.

4. **iCloud corrompe a working tree** (repo está no ~/Desktop). Nunca `git add -A`
   nem `commit -a`; sempre conferir `git status` antes. `node_modules` some →
   `npm install`.
